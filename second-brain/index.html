What I built

The pieces that make memory actually work

🔍
Unified search in one call
megamem query() searches Obsidian, Neo4j, and ChromaDB simultaneously and merges the results. No need to know which database to query.
⏳
Memory decay like a real brain
Neo4j entities have temporal decay — old, unchecked facts gradually fade. The system models memory the way humans actually remember things.
📐
1024-dim semantic vectors
Notes are embedded with qwen3-embedding-8b via OpenRouter. Searching "finance career goals" also finds "MBA salary targets".
🔗
Knowledge graph with relations
Neo4j stores how concepts relate — "SAP BTP" connects to "C_LCNC_2406 cert", which connects to "Naukri job listings". Graph traversal, not just keyword search.
⚡
In-memory index for speed
The 2nd Brain server pre-builds an in-memory index of all 10K+ notes at startup. Searches that took 9 seconds now take milliseconds.
📓
Works without Obsidian running
obsidian-fast reads the vault directly from disk. No need to keep the app open. Browse, read, and append notes any time.
