The hook system — 11 lifecycle invocations

9 Python scripts wired to Claude Code's settings.json — fire automatically, zero human intervention

Session Lifecycle
Session Start
session_start.py
Reads active-context file, restores Redis state (goal, domain, tool count). If context is stale (>24h), pulls from Memento knowledge graph. AI has full context from the very first message of a new session.
Session Stop
stop_archiver_v2.py
Detects domain from session content, writes active-context-{domain}.md snapshot to disk, saves goal + tool count to Redis, compiles knowledge journal into blueprint files. The AI's automatic "end-of-day" handoff.
Edit Safety (fire on every file write)
After Edit/Write
mid_session_checkpoint.py
Increments an edit counter in Redis. Every 15 edits — automatic git commit with chore(checkpoint): prefix. Session crashes, power cuts, and context compaction can never cause lost work. B43
After Edit/Write
gap_check.py
Sets a Redis TTL flag after every file write. AI must run the B33 quality checklist for that file type (Python / Dockerfile / SQL / shell) before moving to the next task. Engineering standards enforced automatically. B33
After Edit/Write
log_change.py
Logs every file change to the hook_audit table in homelab.db — timestamp, file path, tool used, session correlation ID. Full queryable audit trail of everything the AI touched.
Self-Monitoring — Phase 3 (shipped 2026-05-31)
All tools
goal_drift_hook.py
Every 10 calls: extracts keywords from active goal in Redis — checks if recent tool inputs contain them. If 80%+ absent — emits [DRIFT]. Also classifies topic per 10-call batch — 3 different topics in a row — emits [STACK] rabbit-hole warning. B45
Before Read/Grep
token_budget_hook.py
Tracks read-heavy tool calls in a rolling 2-minute window via Redis TTL counter. More than 10 reads — emits [B40] nudge to stderr. Enforces "retrieve less, reason more" discipline. B44
All tools (observer)
session_observer.py
Records every tool call into a Redis list (last 20 calls, TTL 24h). Feeds the goal drift detector. Builds a rolling picture of what the AI is actually doing — queryable at any point during the session.
File Safety
Before file edit
file_lock.py
Acquires a Redis distributed lock before writing a file. Prevents concurrent writes if multiple Claude sessions are running on the same codebase simultaneously.
After file edit
file_lock.py
Releases the Redis lock immediately after the write. If the edit fails or the process dies mid-write, the lock auto-expires via TTL — no deadlocks, no hung sessions.

Phase 3 — the AI monitors itself

Three self-consciousness hooks that make the agent aware of its own behaviour in real time

Phase 3a · Goal Drift Detector
🎯
Am I still working on what matters?
[DRIFT] 80% of goal keywords absent in last 20 calls
Every 10 tool calls, the hook extracts keywords from the active goal stored in Redis and checks if recent tool inputs contain them. If they diverge, the agent gets a stderr alert before drifting further. 10-minute cooldown prevents alert fatigue.
Phase 3b · Rabbit-Hole Detector
🐰
Have I been jumping between topics?
[STACK] Topic shifted 3× in a row: file → web → infra
Each 10-call batch is classified into a topic (file / web / code / memory / infra / other). Three consecutive batches with different topics = rabbit-hole warning. Prevents accidental deep dives that derail the main task. 30-minute cooldown.
Phase 3c · Token Budget Enforcer
📊
Am I over-fetching context?
[B40] Read #11 in 2-min window — check session cache first
A rolling 2-minute window tracks every Read, Grep, Glob, WebFetch call via a Redis TTL counter. More than 10 in the window = nudge fires to stderr. Context is expensive — this enforces disciplined retrieval every single turn.

Hook audit trail — every invocation logged

A live health dashboard runs before any harness session

📊

homelab.db → hook_audit table (+ JSONL sidecar)

Every hook invocation writes: session_id · correlation_id · ts_start · ts_end · duration_ms · hook · hook_type · surface · tool · file_path · event (fired / skipped / degraded / error) · signal · stdout_bytes · error

Run hook_health.py to get a live dashboard: total invocations, error rate, p50/p95 latency per hook, signals fired ([B40] / [DRIFT] / [STACK]), and unclosed sessions (crash indicator). Three surfaces tracked: cognitive (affects AI decisions) · operational (writes files/DB/git) · contextual (reads state only).

Sample behavioral rules

A taste of what 46 rules look like in practice — from B1 silent reads to B46 harness health checks

B8 · Silent Execution
Never ask permission for reads
DB queries, file reads, web searches all execute silently — zero "shall I proceed?" round-trips. Saves a full interaction on every read operation across thousands of tool calls per week.
B13 · Three-Tier Writes
Risk-proportional confirmation gates
New file = silent. Overwrite existing = confirm once. Delete, force-push, or send email = hard stop, no override. The agent can never accidentally destroy or leak data.
B18 · L0-First Routing
Free local tools before LLM calls
For any code or document task, run tree-sitter AST outline first. Only escalate to an LLM API call if the local tool is insufficient. Saves 4–8× tokens on code navigation alone.
B31 · 5-Tier Doc Chain
Never pay for docs you already own
Redis hot cache → ChromaDB → package registries → context7 → Tavily (last resort). Each tier caches the result. Tavily billing is essentially zero in practice after the cache warms up.
B36 · Architecture Doc Check
Read docs before recommending any tech
Before recommending any technology in a design plan, fetch current docs and verify protocol/OS compatibility. This rule was added after a real 6-hour incident. Every failure is codified so it cannot repeat.
B43 · Mid-Session Checkpoint
Auto-commit every 15 edits
PostToolUse hook counts edits in Redis. Every 15 → automatic git commit. Session crashes, context compaction, and force-quits can never cause lost work. The AI is its own safety net.
