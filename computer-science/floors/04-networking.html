Networking & the Internet

One computer is useful. Billions of computers that can talk to each other changed civilisation. This floor is the set of agreements that let any machine reach any other — and, today, the wiring that lets thousands of computers act as one to train an AI.

🌐 What this floor is

Networking is a stack of shared protocols — rules every computer agrees to follow so a message sent by one is understood by another. Those rules cover how to break a message into pieces, how to address it, how to find the destination, and how to rebuild it on arrival. Get everyone to agree, and a phone in Bengaluru can reach a server in Virginia in a heartbeat.

The big ideas

How isolated computers find each other, exchange data, and act as one.

📦 1. Packet switching

Instead of one continuous wire between sender and receiver, the internet chops every message into small numbered packets.

Imagine mailing a **1,000-page book** to a friend, but the post office only allows letters. So you tear out every page, put it in its own stamped envelope with a page number, and mail all 1,000 envelopes separately. Some might travel by train, others by truck. They will arrive at your friend's house out of order, but your friend can easily use the page numbers to arrange them back into a book.

🔎 Where you meet it

A video call surviving a shaky connection — freezing for a second, then catching up — is packet switching: lost packets are simply re-sent, while the rest keep flowing.

The real need

Research computers were rare and hugely expensive, and ARPA wanted scattered labs to share them. There was also a Cold-War wish for communication that could survive parts of the network being destroyed.

What it forced

Packet switching — chopping messages into pieces that each find their own route — so no single broken link could stop the conversation.

🤝 2. The common language: TCP/IP and DNS

For a global network, everyone needs the same rules. TCP/IP — adopted across the ARPANET on 1 January 1983 — became that universal language. It splits communication into **four layers**: the *Link Layer* (the physical wires or Wi-Fi waves), the *Internet Layer* (IP addressing and routing packets across networks), the *Transport Layer* (TCP, which numbers packets to ensure they arrive in order and re-sends any that get lost), and the *Application Layer* (such as HTTP or DNS, which translates raw data into what you see on a screen).

But numbers are hard to remember, so the Domain Name System (**DNS**) was introduced. DNS is the **internet's phone book**. Humans remember names (like youtube.com), but computers only understand phone numbers (IP addresses like 142.250.190.46). DNS does the translation in a millisecond.

🔎 Where you meet it

Every time you type a website name instead of a string of numbers, DNS does an instant lookup in the background. It has quietly performed this translation for you since 1983.

The real need

Early on, every computer kept one shared file, HOSTS.TXT, listing every machine's name and number — copied around by hand. As the network grew, that file stopped scaling.

What it forced

DNS (1983) spread the lookup across thousands of cooperating servers, so the internet could keep adding names forever without one file falling over.

🕸️ 3. The Web, and then the cloud

The internet is the road; the World Wide WebInvented by Tim Berners-Lee at CERN — proposed in 1989 and opened to the public on 6 August 1991. The system of linked pages that runs on top of the internet. is what most people drive on it. Proposed by Tim Berners-Lee at CERN in 1989 and opened to the public in 1991, it added linked pages and addresses (URLs).

Later, the cloud appeared. Instead of buying physical server racks, companies rent virtualized machines on-demand (first popularized by AWS in 2006). But managing thousands of containerized apps across these rented servers by hand was impossible. In 2014, Google open-sourced **Kubernetes**—a container orchestration engine. Think of Kubernetes as the **conductor of a massive server orchestra**. If one server goes down, it automatically reschedules the container to another server; if traffic spikes, it spawns copies of the app to share the load, keeping web services running uninterrupted.

The real need

CERN's scientists used many incompatible computers and kept losing track of each other's documents and data.

What it forced

Berners-Lee invented one simple way to link information across all of them — pages, links and URLs. That is the World Wide Web.

⚡ The 2017 connection

A modern AI model is far too big for one computer. It is trained across thousands of GPUs wired together, constantly exchanging huge bundles of numbers. After 2017, data-centre networking and the cloud were rebuilt around moving that flood of data fast enough — networking became a core part of how AI is even possible.

🎮 Try it yourself: Trace a Web Click

When you click a button to view a photo of a cute puppy online:

  1. 📞 DNS Lookup: Your browser asks the internet's phone book: "Where is puppyphotos.com?" DNS replies: "Go to 14.21.8.99."
  2. 📦 Packet Packing: Your browser wraps your request ("Send me the puppy photo") into packets and mails them over the network.
  3. ⚡ Cloud Delivery: The server at that address opens the envelopes, grabs the puppy photo file, chops it into 50 packets, and mails them back to your computer. Your screen stitches them together in a flash.

How this floor evolved

From a four-computer military test to a worldwide elastic cloud.

Then

1969: a handful of university computers linked by ARPANET, exchanging the first fragile packets.

Now

Billions of devices, one cloud

The whole planet is connected, and computing power is rented on demand from vast data centres anywhere on Earth.

1969
ARPANET's first message ↗

On 29 October 1969 the first host-to-host message travelled over a packet-switched network — the moment the internet's story begins.
1983
TCP/IP becomes standard ↗

TCP/IP becomes the internet's universal language on 1 Jan 1983, standardizing how different networks package and route data.
1983
DNS is introduced ↗

In November 1983, the Domain Name System is created to translate human-readable names like google.com to IP addresses.
1989
The World Wide Web ↗

Tim Berners-Lee proposes the Web at CERN in 1989; it opens to the public in 1991 and becomes how the world uses the internet.
2006
The Cloud (AWS) ↗

Amazon launches EC2 and S3, turning physical server hardware into a virtual software utility rented on demand.
2014
Kubernetes — the cloud OS ↗

Google open-sources Kubernetes, standardizing how containerized server programs are scheduled and managed across large server farms.
2026
Nvidia Spectrum-X plain Ethernet ↗

NVIDIA captures the #1 spot in data center Ethernet switching with Spectrum-X, standardizing massive AI clusters at Meta, Oracle, and xAI over standard Ethernet rather than complex, proprietary fabrics.

⚡ This floor's role in the 2017 cascade

Floor 4 turned many machines into one. Training a Transformer-based model is too big for any single computer, so the network itself became part of the processor — thousands of GPUs exchanging gradients across high-speed links. The cloud made that power rentable by anyone. Without this floor, the 2017 idea would have stayed a clever paper; with it, the idea could be scaled to planet size.
