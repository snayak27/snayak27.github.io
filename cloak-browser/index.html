"Imagine having a disguise that makes you look like a different person every time you walk into a store — but your membership card still works, and the staff still recognises you."

Most automation tools get blocked immediately. Websites detect that a robot is visiting — no cookies, fresh browser, no history, suspicious fingerprint — and throw up a CAPTCHA or ban the account.

CloakBrowser Manager maintains persistent browser profiles with real cookies, real browsing history, real fingerprints. Log into Naukri or LinkedIn manually once through a noVNC screen. After that, automated research sessions use the same profile — websites see a returning human visitor, not a bot.

In technical terms: isolated browser profiles via CloakBrowser Manager, CDP automation through Crawl4AI, noVNC remote desktop for manual login, profile state persisted across all sessions.

What I built

🖥️
noVNC remote desktop
A browser UI at 127.0.0.1:8181 shows the live browser. You manually log into any site, solve any CAPTCHA, establish the session — just once.
🔗
CDP port for automation
After manual login, Crawl4AI connects via Chrome DevTools Protocol at ws://127.0.0.1:9222. Automation uses the real, already-logged-in profile.
💾
Profile persistence
Browser profiles survive container restarts. Cookies, session tokens, and site preferences are preserved. One login, unlimited sessions.
🔍
Profile recovery
If a profile ID becomes stale, GET /api/profiles lists all profiles. IDs are stored in memory and auto-updated when recovered.
