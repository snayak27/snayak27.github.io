How the pieces connect

12 active data flows — every arrow does real work on every session.

BraveBridge→URL Index

Every URL visited in Brave is harvested every 60 seconds and appended to the local SQLite index. The knowledge base grows passively while you browse.

60s harvest interval · 90-day history backfill on setup
URL Index→2nd Brain

Crawled page content is chunked and embedded into ChromaDB (1024-dim), making external web knowledge semantically searchable alongside personal notes.

Crawl4AI async crawl → embed → 2nd Brain search
2nd Brain→UAC

UAC reads cross-session goals, tool history, and active domain from the 2nd Brain's SQLite tables — giving Claude 48h continuity without re-prompting.

48h cross-session memory · 7 Redis state keys
UAC hooks→Every tool call

11 Python hooks fire on PreToolUse and PostToolUse for every single tool call — gap checks, token budget, goal drift detection, checkpoint commits, audit log.

~80% fewer redundant API calls · 0 manual intervention
Behavioral Rules→UAC hooks

46+ behavioral rules define the spec. UAC hooks are the enforcement layer. Rules say "what to do"; hooks make it happen automatically every session.

46+ rules → 11 hooks → 0 manual re-prompting
Claude Code→Claw Router

Any output >30 lines is delegated to Claw Router instead of generated inline. Claude's context stays clean. Cost drops 90%. Quality improves via benchmark routing.

$0.38/M for writing · DeepSeek V3.2 · auto 15-day review
Hermes→Claw Router

Hermes' scheduled generation tasks — daily notes, quiz content, job summaries — route through Claw Router for benchmark-optimal model selection.

Starts at login · 0 triggers · outputs to 2nd Brain
HIVE Gateway→All 34 tools

Claude makes one SSE connection to HIVE at port 8765 and gets access to all 34 MCP tools across all 4 rings — no individual server management.

Hot-reloaded manifests · add service without restart

Pi-hole→All Docker containers

Every Docker container on pihole-net resolves DNS through Pi-hole + Unbound. 0 third-party DNS providers see queries. Watchtower auto-updates all images.

Quad9 failover · auto-updates · network-level ad blocking

The B31 doc-fetch chain checks the URL Index for cached library docs before spending tokens on Tavily or context7 — the free local tier always hits first.

Design philosophy

Three principles that explain why this ecosystem looks the way it does — and why an interviewer should care.

🔗
Everything talks to everything — value compounds

No project is a dead end. BraveBridge feeds URL Index feeds 2nd Brain feeds UAC feeds Claude Code. The value isn't in any single project. It's in the connections. The more the system is used, the smarter it gets — passively, automatically.

⬆️
Data flows up. Control flows down.

Infrastructure serves automation. Automation collects for intelligence. Intelligence informs the platform. The platform serves the orchestrator. Signals (URLs, notes, tool history) flow upward. Commands (navigate, scrape, generate, remember) flow downward. Clean separation.

💰
$0 cloud dependency — local tier always first

Every layer has a free local option that runs before any paid external service. Redis before Chroma before Tavily. BraveBridge before CloakBrowser. Claw Router picks the cheapest benchmark-winning model. Cloud is the last resort — not the default. The whole ecosystem runs on one developer's homelab.
