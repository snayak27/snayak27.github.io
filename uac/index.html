UAC gives the AI a persistent nervous system. It knows what you were working on yesterday. It knows which documentation turned out to be wrong. It knows which programming language kept failing — and avoids it next time.

In technical terms: Redis-backed in-session and cross-session state, three self-correcting feedback loops, Python hooks that fire automatically at session start and stop.

The three feedback loops

How the AI learns from its own mistakes — without being told

1
Doc-Suspect Loop — "That documentation was wrong"
When code written using cached docs fails at runtime, the AI marks that doc as suspect. Next session it skips the cache and re-fetches fresh documentation from source — then updates so the error can't repeat.
redis SET agent:lib_docs:{lib}:{ver}:suspect 1 EX 3600
2
Language-History Loop — "That approach kept failing"
When a programming language choice succeeds or fails in a domain, the outcome is recorded. After 2+ failures for the same domain, the AI automatically avoids it — even if GitHub stars suggest it's popular.
redis HMSET agent:lang_history {domain}:{lang} "failure|N"
3
Goal-Continuity Loop — "Continuing from where we left off"
At every session end, the current goal and domain are saved to Redis. Next session start, if less than 48 hours have passed, the AI automatically resumes the same context — no re-explaining needed.
redis HSET agent:session:latest goal "..." last_active "..."

Phase 3 — the AI watches itself

Three new self-monitoring hooks added 2026-05-31 — the AI now catches its own drift in real time

🎯
Goal drift detector (B45)
Every 10 tool calls, a hook checks if recent actions still match the active goal keywords stored in Redis. 80%+ absent → emits [DRIFT] signal to stderr before the session wanders further.
🐇
Rabbit-hole detector (B45)
Each 10-call batch is classified by topic. Three different topics in a row → emits [STACK] warning. The AI notices when it has been pulled down unrelated threads and self-corrects.
📊
Token budget enforcer (B44)
Tracks read-heavy calls in a rolling 2-minute window via Redis TTL key. More than 10 reads → emits [B40] nudge. Enforces "retrieve less, reason more" discipline automatically.
🗂️
Hook audit trail (B46)
Every hook invocation is logged to homelab.db with timing, surface type (cognitive/operational/contextual), and outcome. Run hook_health.py for a live dashboard before any harness session.

What I built

🔁
Session hooks (start + stop)
Python scripts fire automatically when Claude Code starts and stops. They write state to Redis so context is always preserved, even on unexpected shutdowns.
🌐
UAC Daemon (FastAPI)
A lightweight service at port 7001 with endpoints for ecosystem resolution, feedback submission, and nightly housekeeping. Accessible directly via HTTP — no docker exec needed.
📚
5-tier doc-fetch chain
Before using any library, UAC checks Redis → ChromaDB → package registries → context7 → Tavily. Each tier caches results to prevent repeat API calls.
🌍
Ecosystem-first language selection
Before writing code, UAC checks GitHub for which language the ecosystem actually uses — Go, Rust, TypeScript, or Python — based on real repos, not assumptions.
