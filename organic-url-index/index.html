Organic URL Index

The AI's Personal Google

A web crawler and RAG data pipeline for AI agents — cache-first, content-addressed storage with 4-tier search that costs near-zero after warmup and grows smarter with every query.

Web CrawlerRAG Data PipelineSemantic SearchContent-Addressed StoragePythonSQLiteCrawl4AI

Imagine this…

A story a 10-year-old would understand

"Imagine if Google remembered every page you ever visited — and could find it instantly, without asking the internet again."

Every time an AI agent searches the web, it pays — in time, API credits, or rate limits. And it forgets the result immediately after.

The Organic URL Index works like a personal library that grows with every search. First check your own shelf. If it's there, return it in milliseconds. If not, go fetch it and add it to the shelf for next time.

In technical terms: 4-tier search chain (Redis → FTS5 SQLite → SearXNG → Tavily). Content-addressed storage using SHA-256. Piece-based parallel crawls inspired by BitTorrent.

The 4-tier search chain

Every search checks cheaper sources before expensive ones

T0
Redis Cache
In-memory hot cache. Exact URL lookups return in under 1 millisecond. Most repeated agent queries hit here first.
⚡ <1ms · zero cost
T1
SQLite FTS5 Full-Text Search
Local database with 161+ indexed URLs and crawled content. Returns results with a 200-char content preview from previously-crawled pages.
🔍 ~5ms · zero cost · includes content preview
T2
SearXNG Self-Hosted Search
A private meta-search engine running in Docker. Aggregates results from multiple search engines without sharing your queries with anyone.
🌐 ~200ms · private · no API cost
T3
Tavily API — last resort only
Called only when all local tiers miss. Result is immediately stored in SQLite and Redis to prevent the same query hitting Tavily again.
💰 ~800ms · billed · cached after first hit

The BitTorrent connection

Three ideas borrowed from peer-to-peer file sharing

BitTorrent concept
Content Hashing
↓ adapted as
SHA-256 content identity
The same article on two different URLs gets stored once. The hash is the identity — not the URL.
BitTorrent concept
Piece-based downloading
↓ adapted as
Piece-based crawl jobs
Bulk crawls split into pieces of 50 URLs. Failed pieces retry independently — completed pieces never re-crawled.
BitTorrent concept
Mirror routing
↓ adapted as
Smart URL rewriting
arxiv.org → ar5iv.org, Wikipedia → REST API, GitHub blob → raw. Fetches faster mirrors automatically.
BitTorrent concept
Seeder backpressure
↓ adapted as
Agent backpressure signal
When the AI is actively searching, crawl workers pause 30s. Prevents CPU spikes during interactive sessions.
